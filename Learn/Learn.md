# 💀 VOID-Core: Официальное руководство разработчика (v1.0)

**VOID-Core** — это мультипарадигмальный системный DSL, созданный для разработки защищенных распределенных систем. Он объединяет производительность **C++**, безопасность **Rust**, лаконичность **Python**, асинхронность **JS (Node.js)** и нативную автоматизацию **Playwright**.

---

## 1. Основы архитектуры (Концепция Двух Миров)

Программа на VOID-Core всегда состоит из двух типов файлов, которые компилятор «сплавляет» в один бинарник:

1.  **`.ctos` (Core Tactical OS)** — Низкий уровень (Ядро). Здесь ты управляешь железом, памятью и защитой.
2.  **`.vc` (Virtual Component)** — Высокий уровень (Приложение). Здесь ты пишешь бизнес-логику, веб-ботов и интерфейс.

---

## 2. Синтаксические маркеры (Контексты)

Внутри любого файла ты должен явно указывать компилятору, в каком «режиме» он работает. Это делается с помощью маркеров:

*   `//---- [SYS] ----/` — Режим **C++/Rust**. Прямой доступ к памяти, регистрам и прерываниям.
*   `//---- [LOGIC] ----/` — Режим **TS/Python**. Асинхронность, высокоуровневые объекты и сборщик мусора.
*   `//---- [VIEW] ----/` — Режим **React/JS**. Описание графического интерфейса.

---

## 3. Переменные и Тип данных `CELL`

В VOID-Core нет обычных типов. Есть **CELL** — универсальный контейнер, который меняет свои свойства.


| Тип | Как писать | Описание |
| :--- | :--- | :--- |
| **Vocal** | `cell name = "data"` | Обычная переменная (как в Python/JS). |
| **Raw** | `cell:raw ptr = 0x44` | Прямой адрес в памяти (как в C++). |
| **Ghost** | `cell:ghost key = "..."` | **HARDCORE:** Данные шифруются в RAM ключом CPU. |

## 4. Обучающий пример: "Скрытый Сетевой Узел"

Давай напишем программу, которая проверяет HWID устройства, и если всё ок — запускает скрытый браузер.

### Шаг 1: Пишем Ядро (`security.ctos`)
Здесь мы создаем "Броню".

```cpp
//---- [SYS] ----/
node Shield {
    // Данные, которые невозможно прочитать дампом памяти
    cell:ghost my_hwid = "DELL_9300_NG_8821";

    pub entry verify() {
        // 1. Проверка железа
        if (sys.hwid() != my_hwid) {
            sys.trigger_fake_panic("0x0000005"); // Симуляция ошибки
        }
        
        // 2. Блокировка BadUSB (Игнорируем программный ввод)
        interrupt.set_filter(mode: PHYSICAL_ONLY);
        
        // 3. Политика Zeroize (Стирать всё после выхода)
        mem.set_policy(AUTO_ZEROIZE);
    }
}

### 5. Пишем Приложение (app.vc)
Здесь мы используем TypeScript и Python стиль.
typescript

//---- [LOGIC] ----/
// Оператор &- создает мост к ядру
&- import { Shield } from "security.ctos";

app_entry {

    async let browser = web.launch({ stealth: true });

### 6. Основные команды (Мануал)
Работа с памятью (Rust/C++ Style)

    move a -> b; — Передача владения данными (старая переменная a затирается).
    mem.zeroize(target); — Принудительное физическое уничтожение данных в RAM.

Сеть (Stealth Style)

    net.raw_inject(packet, "bypass_vpn"); — Пробив VPN на уровне драйвера.
    telemetry.report_origin("bypass"); — Скрытие реального IP в ядре языка.

Подпрограммы (Linux Style)

    spawn isolate MyDaemon { ... }; — Создание независимого процесса внутри твоей программы, который живет своей жизнью.

### 7. Как компилировать и запускать

    Сборка: Используй voidc (твой компилятор).
    bash

    voidc app.vc --target psp_ng

    Use code with caution.
    Запуск: Используй voidrun (твой рантайм).
    bash

    voidrun launcher.ng

    Use code with caution.

### 8. Важные советы

    Никогда не пиши cell:vocal для паролей. Только cell:ghost.
    Если программа должна "исчезнуть", вызывай sys.self_destruct(). Это сотрет все ключи в RAM и закроет процесс без следов в логах ОС.
    Используй //---- [VIEW] ----/ только для вывода статусов. Вся тяжелая работа должна быть в [LOGIC], а защита — в [SYS].
