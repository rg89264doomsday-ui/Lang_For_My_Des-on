Справочник типов и системных сущностей VOID-Core (v1.0)
Этот документ является обязательным дополнением к основному мануалу. Он описывает, как данные физически существуют в памяти и как ими управлять.
1. Универсальный тип cell и его модификаторы
В VOID-Core нет разделения на int, string или bool на уровне синтаксиса. Всё есть ячейка (cell), но её поведение определяется модификатором.
Модификатор	Синтаксис	Слой	Описание (Поведение в RAM)	Язык-аналог
Vocal	cell:vocal	[LOGIC]	Открытые данные. Динамическая типизация. Авто-очистка (ARC).	Python / JS
Ghost	cell:ghost	[SYS]	Шифрование AES-NI. Данные расшифровываются только в регистрах CPU.	VOID-Security
Raw	cell:raw	[SYS]	Прямой адрес (Pointer). Без проверок границ. Максимальная скорость.	C++ / ASM
Secure	cell:sec	[LOGIC]	Типизированная ячейка. Проверяется Borrow Checker'ом.	Rust
2. Операторы управления данными (Data Ops)
Эти операторы определяют, как информация перемещается между слоями и подпрограммами.

    &- (Bridge) — Связывает высокоуровневую логику с низкоуровневым ядром.
        Пример: &- call Kernel.wipe();
    >> (Pipe) — Передает поток данных из одного isolate в другой (Linux-style).
        Пример: data_stream >> NetworkNode;
    move — Передача владения. Старая переменная мгновенно подвергается Zeroize.
        Пример: move secret_key -> Vault;
    ghost_cast<T> — Принудительное наложение шифрования на обычную ячейку.

3. Системные узлы (Native Nodes)
Это встроенные в язык объекты, которые не требуют импорта библиотек.
sys — Управление системой и Anti-Forensics

    sys.hwid() — Возвращает уникальный хеш железа (CPU + Motherboard).
    sys.trigger_fake_panic(hex_code) — Вызывает имитацию BSOD или Kernel Panic.
    sys.anti_debug_score() — Возвращает уровень риска (0-255) присутствия отладчиков.
    sys.self_destruct() — Стирает ключи в RAM и завершает процесс без логов.

mem — Управление памятью

    mem.set_policy(AUTO_ZEROIZE) — Политика очистки RAM при выходе из функции.
    mem.physical_wipe(addr, size) — Прямая перезапись физического участка RAM нулями.
    mem.lock_pages() — Запрет на сброс памяти в SWAP (файл подкачки на диске).

net — Скрытая сеть (Stealth Net)

    net.raw_inject(packet, mode) — Инъекция пакетов мимо системного стека (Bypass VPN).
    net.listen_stealth(port) — Открытие порта, невидимого для netstat и сканеров.

web — Нативный Playwright (Automation)

    web.launch({ stealth: true }) — Запуск движка без GUI и следов в системе.
    web.proxy_bypass() — Использование встроенного в язык механизма маршрутизации.

4. Конструкции управления (Flow Control)
spawn isolate Name { ... }
Создает независимую подпрограмму (актор). У каждого изолята своя куча памяти и свой PID. Если один изолят падает — вся система продолжает работать.
scoped_shield(policy) { ... }
Блок кода, внутри которого действуют особые правила безопасности.

    Пример: scoped_shield(on_input: ignore_injected) { ... } — Внутри этого блока BadUSB бессилен.

[x for x in collection if condition]
Встроенный в синтаксис механизм обработки данных. Компилируется напрямую в векторные (SIMD) инструкции процессора для обработки гигабайтов трафика или логов.
5. Спецификация файлов (.ctos vs .vc)
Расширение	Назначение	Основной Слой	Привилегии
.ctos	Драйверы, Ядро, Защита	//---- [SYS] ----/	ROOT / RING 0
.vc	Приложения, Боты, UI	//---- [LOGIC] ----/	USER / SANDBOX
